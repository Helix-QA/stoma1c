Справочник "Обращения"

Справочник предназначен для хранения списка обращений потенциальных пациентов (лидов). Лид - потенциальный пациент, тем или иным образом отреагировавший на маркетинговую коммуникацию. Цель получения лидов - последующая отработка взаимодействия с потенциальным пациентом или продажа базы лидов покупателям информации о потенциальном интересе целевой группы к тому или иному предложению.

В данный справочник вносится следующая информация:

  • Фамилия, имя, отчество (ФИО)- указывается фамилия, имя и отчество потенциального пациента. На основании ФИО программа автоматически формирует представление потенциального пациента.
  • Контакты заявки. Группа контактных данных потенциального пациента:
    • Телефон - основной телефон потенциального пациента.
    • Email - электронный адрес потенциального пациента. По нажатию на кнопку  можно отправить письмо непосредственно из программы.
    • Адрес - адрес проживания потенциального пациента. Используется при печати договора.
    • Рассылка разрешена (рассылка запрещена) - по нажатию на ссылку происходит переключение флага разрешения отправки потенциальному пациенту смс-рассылок.
  • Группа "Маркетинг". В группе маркетинг представлены данные, позволяющие указать тот способ, благодаря которому пациент узнал о клинике:
    • Канал привлечения - ссылка на канал привлечения  пациента.
    • Рекламный источник - в данном поле необходимо указать откуда пациент узнал о клинике. Можно указать в качестве источника другого пациента клиники. В этом случае необходимо будет указать этого пациента, выбрав его из справочника пациентов.
  • Куратор - выбирается из справочника сотрудников. Заполняется автоматически текущим пользователем программы при создании заявки.
  • Группа "Параметры UTM". UTM-метки - специализированные параметры в URL, используемые маркетологами для отслеживания рекламных кампаний в сети Интернет:
    • utm_source - идентифицирует сайт, с которого отправляется трафик. Обязательный параметр.
    • utm_medium - идентифицирует рекламную модель. Примеры: ссылка cost per click, сообщение электронной почты.
    • utm_campaign - идентифицирует конкретную рекламную кампанию.
    • utm_term - идентифицирует поисковые термины, если переход был из поиска..
    • utm_content - идентифицирует конкретный элемент контента, на который кликнул пользователь перед переходом на сайт - например баннерное объявление или текстовую ссылку.
  • Дополнительные реквизиты. В данной группе находятся дополнительные реквизиты, созданные пользователем.
